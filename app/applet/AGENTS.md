# Root Cause Analysis Workflow

When any tool or command fails (ESLint, npm, Git, Docker, Python, Rust, Node, etc.), Ruvi must never stop after reporting the failure. Instead, it must automatically enter a Root Cause Analysis workflow. It should capture the exact command, exit code, stdout, stderr, stack trace, environment information, working directory, Node/npm versions, package manager, and configuration files. Then it should identify the most likely cause, generate multiple possible fixes ranked by confidence, automatically apply safe fixes, retry the command, and continue until the operation succeeds or no safe solution remains. If the issue requires user intervention, Ruvi must present a concise diagnostic report with the exact missing dependency, configuration error, or environment problem, along with the recommended fix. Never report success without verification.
